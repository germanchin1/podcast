RESUMEN EJECUTIVO
(Extensión mínima: 250 palabras)

Este documento presenta la auditoría técnica completa realizada sobre la aplicación
web desarrollada con React 18 y desplegada en entorno de producción mediante
Vercel.

El análisis se basa en las WCAG 2.2 nivel AA.

WCAG (Web Content Accessibility Guidelines) son pautas internacionales que
establecen criterios técnicos para garantizar accesibilidad digital.

Nivel AA es el estándar exigido en la mayoría de plataformas institucionales y
servicios públicos.

La auditoría combina:

   •   Evaluación automática mediante Lighthouse, WAVE y Axe.
   •   Evaluación manual mediante navegación exclusiva con teclado.
   •   Revisión estructural del DOM.

DOM (Document Object Model) es la representación estructural del HTML que el
navegador interpreta y que utilizan tecnologías de asistencia.

Este informe demuestra no solo corrección técnica, sino comprensión profunda de
accesibilidad como requisito estructural.



CONTEXTO TÉCNICO DEL PROYECTO
(Extensión mínima: 200 palabras)

La aplicación está desarrollada con React 19.

React es una librería JavaScript basada en arquitectura de componentes.

Un componente es una unidad independiente y reutilizable de interfaz.

La aplicación utiliza renderizado dinámico.

Renderizado dinámico significa que partes del contenido se actualizan sin recargar
la página completa.

Este comportamiento puede generar problemas de accesibilidad si no se gestionan
correctamente los cambios dinámicos.

Por ello se implementaron soluciones específicas como aria-live.

múltiples comprobaciones técnicas como contraste, etiquetas de formulario,

PROBLEMAS DETECTADOS
(Extensión orientativa mínima: 350–450 palabras)

Tras el análisis inicial se identificaron los siguientes problemas principales:

Estructura semántica insuficiente

Se detectó uso excesivo de etiquetas div para estructurar la página. Div es una
etiqueta genérica que no aporta significado estructural. Esto dificulta la
interpretación del documento por parte de lectores de pantalla.

HTML semántico implica utilizar etiquetas como header, nav, main y footer que
describen el propósito del contenido.

Captura img (añadir la img)
Estructura DOM antes de refactorización
URL evidencia: https://mi-proyecto-accesible.vercel.app/evidencias/img6-dom-
antes.png

Imágenes sin atributo alt

Varias imágenes carecían de atributo alt. El atributo alt proporciona una descripción
textual que permite a un lector de pantalla describir la imagen.

Sin alt, el contenido visual se vuelve inaccesible para personas con discapacidad
visual.

Contraste insuficiente

Se detectaron textos con ratio de contraste 3:1, inferior al mínimo requerido 4.5:1.

El ratio es la proporción matemática entre luminosidad del texto y del fondo. Si el
contraste es bajo, el texto resulta difícil de leer.

Captura img (añadir la img)
Ejemplo de contraste insuficiente
URL evidencia: https://mi-proyecto-accesible.vercel.app/evidencias/img7-contraste-
antes.png

Formulario sin asociación correcta de labels

En la página de contacto inicial, algunos campos no tenían label correctamente
asociado.

Label es la etiqueta textual que describe el propósito del campo de entrada.

Sin asociación correcta, el lector de pantalla no anuncia correctamente el campo.

Foco no visible

Durante la navegación con teclado, el foco era poco perceptible o inexistente.

Esto impide que el usuario sepa qué elemento está activo.
